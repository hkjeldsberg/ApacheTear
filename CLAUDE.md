## Project Structure

```text
app/              # Next.js App Router — pages, layouts, Server Actions, API routes
components/       # Shared UI — editor/, file-tree/, search/, workspace/, ui/
lib/              # Business logic — db/ (Supabase queries), export/, supabase/, errors.ts
types/            # database.types.ts (generated by supabase gen types)
__tests__/        # Vitest unit + component tests mirroring src layout
```

## Commands

```bash
pnpm dev          # Start dev server
pnpm build        # Production build
pnpm test         # Vitest unit + component tests
pnpm lint         # ESLint
supabase gen types typescript --project-id <ref> --schema public > types/database.types.ts
```

## Code Style

- TypeScript strict — no `any`, explicit return types on all functions
- Server Components by default; `'use client'` only for interactive components
- All DB queries go through `lib/db/` — no raw Supabase calls in components
- All mutations from Client Components go through Server Actions in `app/actions/`
- Files must stay under 300 lines — split into modules when approaching limit
